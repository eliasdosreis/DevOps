# Conceitos de Cobrança AWS

## 1. Analogia do Dia a Dia
Cobrança AWS é como a conta de luz: você paga pelo que consome, com tarifas diferentes para cada tipo de serviço e descontos para uso contínuo ou em grande volume.

## 2. O que é (definição técnica oficial AWS)
A cobrança na AWS é baseada em consumo (pay-as-you-go), com opções de preços sob demanda, instâncias reservadas, savings plans e descontos por volume.

## 4. Questões de Fixação
1. Como funciona a cobrança na AWS?
A) Paga-se uma taxa fixa mensal
B) Paga-se pelo uso dos recursos (correta)
C) Só paga se usar mais de 1TB
D) Não há cobrança

Explicação:
- B) Correta: modelo pay-as-you-go.
- A), C), D) não refletem o modelo AWS.

Armadilha: achar que todos os serviços têm o mesmo preço.

2. O que são Savings Plans?
A) Planos de desconto para uso contínuo (correta)
B) Backups automáticos
C) Serviços de monitoramento
D) Instâncias spot

Explicação:
- A) Correta: Savings Plans são para economia.
- B), C), D) não são Savings Plans.

Armadilha: confundir Savings Plans com instâncias reservadas.

## 5. Conceito Senior
- Savings Plans oferecem flexibilidade maior que instâncias reservadas.
- É possível combinar diferentes modelos de cobrança para otimizar custos.

## 6. Resumo para Revisão
- Cobrança AWS = pay-as-you-go | preços sob demanda/reservado | savings plans | descontos por volume | otimização de custos
