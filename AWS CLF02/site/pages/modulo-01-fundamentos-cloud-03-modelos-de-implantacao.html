Modelos de implantação: Public, Private, Hybrid Cloud

1. Analogia do Dia a Dia

Pense em moradia:

  • Nuvem pública: morar em um prédio (recursos compartilhados, mas separados)
  • Nuvem privada: casa própria (só você usa)
  • Nuvem híbrida: casa própria + uso de hotel quando precisa

2. O que é (definição técnica oficial AWS)

  • Nuvem pública: recursos de TI oferecidos ao público pela internet (ex: AWS, Azure, Google Cloud)
  • Nuvem privada: recursos exclusivos para uma organização, podendo estar no datacenter próprio ou hospedado
  • Nuvem híbrida: combinação de nuvem pública e privada, com integração entre elas

3. Ficha de Estudo Comentada

============================================================ TEMA: Modelos de Implantação DOMÍNIO DO EXAME: Conceitos de Nuvem (24%) ============================================================

O QUE É:

  • Pública: recursos compartilhados, acesso via internet
  • Privada: recursos exclusivos, maior controle
  • Híbrida: integração dos dois modelos

PALAVRAS-CHAVE NA PROVA: → "nuvem pública = AWS" → "nuvem privada = datacenter dedicado" → "nuvem híbrida = integração"

QUANDO USAR: ✅ Pública: agilidade, escala, custo ✅ Privada: requisitos de segurança/compliance ✅ Híbrida: necessidades mistas

QUANDO NÃO USAR: ❌ Pública: dados ultra sensíveis sem criptografia ❌ Privada: se precisa de escala global rápida ❌ Híbrida: se integração é complexa demais

DIFERENCIAIS:

  • Híbrida permite o melhor dos dois mundos

4. Questões de Fixação

  1. O que caracteriza nuvem híbrida?

    2. A) Só recursos locais B) Só recursos públicos C) Integração entre nuvem pública e privada (correta) D) Uso de SaaS

Explicação:

  • C) Correta: integração é a chave.
  • A), B), D) não são híbridas.

Armadilha: confundir híbrida com multi-cloud.

  1. Quando usar nuvem privada?

    2. A) Para escala global B) Para requisitos de compliance rigorosos (correta) C) Para startups D) Para backup simples

Explicação:

  • B) Correta: compliance exige controle.
  • A), C), D) não são motivos principais.

Armadilha: achar que privada é sempre melhor.

5. Conceito Senior

  • Híbrida é comum em grandes empresas migrando para nuvem.
  • Exige integração segura e governança clara.

6. Resumo para Revisão

  • Pública = recursos compartilhados | Privada = controle total | Híbrida = integração dos dois | escolha depende de requisitos de negócio
