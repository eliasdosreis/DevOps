## 🧠 Estilo de Ensino

Cada conceito segue **sempre** esta estrutura:

1. **Analogia do Dia a Dia** — Explica com exemplos do mundo real
2. **O que é (definição técnica Senior)** — Definição precisa como em documentações
3. **Arquivo Jenkinsfile/Script Comentado** — Cada linha explicada
4. **Comandos Passo a Passo** — O que fazer, o que esperar, o que fazer se der erro
5. **Verificação e Troubleshooting** — Como confirmar que funcionou
6. **Conceito Senior** — O "porquê" profundo que Junior não sabe
7. **Pergunta de Entrevista** — Perguntas reais com respostas esperadas

---

## 📁 Regra de Ouro: Arquivos Pequenos e Focados

Cada arquivo tem **um único propósito**. O aprendizado é **lento, preciso e incremental**.

## 🛠️ Pré-requisitos

- Docker instalado (para rodar Jenkins sem poluir a máquina)
- Git instalado
- Editor de código (VS Code recomendado)
- Conexão com internet

## 🚦 Como Estudar

1. Comece pelo **Módulo 0** e siga a ordem
2. Leia o `README.md` de cada módulo antes dos arquivos
3. **Execute cada pipeline** antes de avançar
4. Tente responder as perguntas de entrevista sem olhar a resposta
5. Pratique até conseguir explicar o conceito para outra pessoa
