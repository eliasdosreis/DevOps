# ============================================================
# O QUE ESTE ARQUIVO FAZ:
# Junta a teoria com a pr’┐Įtica: Constr’┐Įi um Deploy completo (MySQL) 
# amarrado no PVC criado anteriormente, garantindo de uma vez por todas
# que nem o k8s matando o pod n’┐Įs perderemos o database.
# ============================================================

# 1. ANALOGIA DO DIA A DIA
# N’┐Įs fomos ao Vendedor "AWS", pedimos um Contrato de lote (PVC) com tamanho de 5G...
# Isso gerou um bloco f’┐Įsico cortado com muro do lado de fora longe da VM.
# AGORA O ’┐ĮLTIMO PASSO ’┐Į amarrar isso no App.
# ’┐Į semelhante a pegarmos aquele gigantesco Pen-Drive m’┐Įgico que compramos,
# e plugar no chassi/bunda do Container/Pod na hora em que ele subir, 
# e criar uma linha de link (Mount) que diz: 
# "Tudo que voc’┐Į Container do DB for salvar na pasta interna nativa C:/mysql/dados..  
# na verdade, voc’┐Į n’┐Įo salvar’┐Į dentro do Docker. Esse cano joga fisicamente os dados DENTRO daquele pendrive!".
# Fechando o Lifecycle.

# 2. O QUE ’┐Į (defini’┐Į’┐Įo t’┐Įcnica Senior)
# Voc’┐Į anexa PVCs ao Pod referenciando o claim exato dentro da declara’┐Į’┐Įo base  
# 'volumes: ' . Isto por sua vez mapeia no 'volumeMounts' das defs do bloco containers.

# ============================================================
# 3. ARQUIVO YAML COMENTADO
# ============================================================
apiVersion: apps/v1
kind: Deployment
metadata:
  name: bd-loja-resistente
  namespace: meu-primeiro-projeto
spec:
  replicas: 1                             # Bancos em Deployments via de regra usam 1 replica limpa
  selector:
    matchLabels:
      app: mysql-persistent
  template:
    metadata:
      labels:
        app: mysql-persistent
    spec:
      containers:
        - name: meu-mysql-container
          image: mysql:8.0                # MySQL Oficial
          env:
            - name: MYSQL_ROOT_PASSWORD
              value: root_master_123      # Em prods reais injete Secrets (Aula 04) aqui!
              
          ports:
            - containerPort: 3306
            
          # O Fio Condutor: Plugando a pasta do Cont’┐Įiner -> ao HD Falso listado no chassi inferior
          volumeMounts:
            - name: disco-falso-do-pvc     # O nome da abstra’┐Į’┐Įo inventada abaixo  !
              mountPath: /var/lib/mysql    # ’┐Į AQUI QUE O ARQUIVO SALVA. Se mudar essa VAR, o mount isola do db e vaza os dados ao reboot!
              
      # No Chassi do Pod isolado as Configura’┐Į’┐Įes GERAIS do HD que ser’┐Įo mapeadas acima:
      volumes:
        - name: disco-falso-do-pvc         # O nome da abstra’┐Į’┐Įo inventada pra ligar na linha 44
          persistentVolumeClaim:           # Tipo de disco: "Busca um Contrato de PVC Registrado na rua"
            claimName: meu-contrato-de-disco-5gb # O RG/Nome Exato do PVC criado na Aula 03.

# ============================================================
# 5. VERIFICA’┐Į’┐ĮO E TROUBLESHOOTING
# ============================================================
# Muitos PV/PVC travam em "Pending" ou o pod falha com "AttachVolume.Attach failed / timeout".
# Principal causa: Voc’┐Į colocou PVC ReadWriteONCE na Cloud. Mas os n’┐Įs das maquinas t’┐Įo cheios. 
# O N’┐Į (A) crashou. Mas o PV por problemas de lat’┐Įncia EC2 ainda constava no Driver CSI amazon
# como 'Apertado/Attached' ao hardware estragado A.
# O Scheduler manda O POD pro n’┐Į sadio (B). O N’┐Į B tenta enfiar o cabo no pendrive por software EC2, e diz
# "Oxe, t’┐Į em uso". E fica em Deadlock infinitamente quebrando seu dia de trabalho. Fique atento.

# ============================================================
# 6. CONCEITO SENIOR (o "porqu’┐Į" profundo)
# ============================================================
# Ao montar o `mountPath: /var/lib/mysql` de um pvc na cloud (Como EBS/AWS ou Disk/Google) voc’┐Į deve notar 
# que o disco por nativo da cloud subir’┐Į FORMATADO num EXT4 LVM normal, possuindo como Default
# o grupo file-permission de chown: `root:root`. 
# Se sua empresa for S’┐Įnior de Seguran’┐Įa usando `securityContext: runAsUser: 999 (NonRoot)` no spec container.
# Voc’┐Į fatalmente ganhar’┐Į crashs Permission Denied. Seniors criam Hooks iniciais de InitContainers chown-ando a 
# rota var/lib/mysql antes do pod subir principal p/ que o mysql tenha perm de escrita na montagem remota!
